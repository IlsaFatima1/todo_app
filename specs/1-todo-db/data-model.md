# Data Model: Todo Application with SQLModel

## Entity: Todo

### Fields
- **id** (Integer, Primary Key, Auto-increment)
  - Unique identifier for each todo item
  - Auto-generated by the database

- **title** (String, Not Null)
  - Title or brief description of the todo
  - Maximum length: 255 characters
  - Required field for all todos

- **description** (String, Nullable)
  - Detailed description of the todo task
  - Maximum length: 1000 characters
  - Optional field

- **completed** (Boolean, Default: False)
  - Status indicator showing if the todo is completed
  - Default value is False for new todos

- **created_at** (DateTime, Not Null)
  - Timestamp when the todo was created
  - Automatically set to current time for new records

- **updated_at** (DateTime, Not Null)
  - Timestamp when the todo was last modified
  - Automatically updated when record is changed

- **user_id** (Integer, Nullable)
  - Foreign key linking todo to a user
  - Used for future authentication-based data isolation
  - Currently nullable to support anonymous todos

### Relationships
- **user** (Optional relationship to User entity)
  - Will be established when user authentication is implemented
  - Currently supports nullable foreign key for backward compatibility

### Validation Rules
- Title must be provided and not exceed 255 characters
- Description, if provided, must not exceed 1000 characters
- Completed status is boolean with default of False
- created_at and updated_at are automatically managed by the system

### State Transitions
- **New Todo**: created_at and updated_at set to current time, completed=False
- **Updated Todo**: updated_at updated to current time, other fields may change
- **Completed Todo**: completed field changed from False to True
- **Reopened Todo**: completed field changed from True to False
- **Deleted Todo**: Record permanently removed from database

## Entity: Database Connection

### Components
- **Engine**: SQLModel engine for database connections
- **Session**: SQLModel session for transaction management
- **Connection String**: Environment-configured Neon PostgreSQL connection
- **Pool Settings**: Connection pooling configuration for performance

### Configuration
- Connection string loaded from environment variables
- Connection pooling for concurrent operations
- Automatic retry logic for transient failures
